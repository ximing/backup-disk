## Codebase Patterns
- Project structure follows standard Go layout: cmd/, pkg/, internal/
- CLI commands use cobra framework, defined in cmd/cloudsync/
- Configuration package in pkg/config/ handles YAML parsing and validation
- Logger package in pkg/logger/ provides structured logging with task-specific outputs
- Environment variables in config are expanded using os.ExpandEnv()
- Config directory: ~/.cloudsync/, logs: ~/.cloudsync/logs/, data: ~/.cloudsync/data/
- State database: ~/.cloudsync/data/state.db using SQLite via modernc.org/sqlite
- Log rotation uses 100MB limit with 5 backup files by default
- State tracking is done via pkg/state with StartExecution/CompleteExecution pattern
- Notification uses pkg/notify with MeoWNotifier, supports always/on-error/never levels
- Retry uses pkg/retry with exponential backoff (1s, 2s, 4s) and error classification
